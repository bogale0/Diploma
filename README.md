# Школа программирования

Это дипломный проект: настольное приложение для изучения основ программирования с теорией, практическими задачами и автоматической проверкой решений. Проект рассчитан на две роли:

- **студент** изучает курсы, читает теорию, запускает код на своих входных данных и отправляет решения на тесты;
- **преподаватель** пополняет платформу: добавляет языки, темы, задания и тесты для проверки.

Идея проекта — дать начинающему разработчику понятный учебный маршрут: выбрать язык, пройти тему, решить задачу и увидеть прогресс. Сейчас в базе заложены курсы по **C++**, **Python**, **C#** и **Go**.

## Что умеет проект

### Для студента

- регистрация, вход и восстановление пароля;
- просмотр списка языков программирования;
- просмотр тем внутри выбранного языка;
- чтение теоретического материала;
- выполнение практических заданий;
- запуск решения с пользовательским вводом;
- отправка решения на автоматические тесты;
- личный кабинет с общим и покурсовым прогрессом.

### Для преподавателя

- отдельная роль `teacher`;
- панель управления учебным контентом;
- создание нового курса/языка;
- создание темы с теорией;
- создание задания;
- добавление тестов к заданию.

## Архитектура

Проект состоит из двух основных частей:

1. **Клиентское приложение** — Qt/QML + C++.
2. **Backend API** — PHP + MySQL.

Клиент общается с сервером по HTTP API. В QML описан интерфейс, а C++-класс `ApiClient` отвечает за сетевые запросы, авторизацию и передачу данных из API в интерфейс.

```text
[Qt/QML desktop client]
          |
          | HTTP JSON API
          v
[PHP backend] ---- [MySQL database]
          |
          v
[проверка и запуск решений]
```

## Как работает пользовательский сценарий

1. Пользователь открывает приложение.
2. На главной странице выбирает язык программирования.
3. Затем выбирает тему.
4. Читает теорию и переходит к заданию.
5. Если пользователь не авторизован, приложение предлагает войти или зарегистрироваться.
6. На странице задания пользователь пишет код.
7. Код можно:
   - **запустить** на собственном вводе;
   - **отправить** на проверку по подготовленным тестам.
8. После успешной проверки backend обновляет прогресс пользователя.
9. Прогресс отображается в личном кабинете.

## Backend API

Backend написан на PHP и возвращает JSON-ответы. Общая логика вынесена в `api/functions.php`:

- подключение к MySQL;
- формирование JSON-ответов;
- проверка Bearer-токена;
- проверка роли преподавателя.

Авторизация работает через Bearer-токен. Токены хранятся в таблице `sessions` и действуют ограниченное время.

Основные группы API:

- **пользователи**: `login.php`, `signup.php`, `recover_password.php`;
- **учебный контент**: `languages.php`, `themes.php`, `theory.php`, `tasks.php`, `task.php`;
- **решения**: `run_task.php`, `check_task.php`;
- **прогресс**: `progress.php`;
- **панель преподавателя**: `create_language.php`, `create_theme.php`, `create_task.php`, `create_test.php`.

## База данных

SQL-схема находится в `api/db.sql`. В ней описаны таблицы:

- `users` — пользователи и их роли;
- `sessions` — активные сессии;
- `languages` — курсы/языки программирования;
- `themes` — темы внутри курса;
- `tasks` — практические задания;
- `task_tests` — тесты для автоматической проверки;
- `user_theme_progress` — прогресс по темам;
- `user_task_solved` — решённые задания.

В `db.sql` также есть стартовые данные: несколько языков, тем, заданий и тестов. Это удобно для демонстрации проекта без ручного наполнения базы.

## Клиентское приложение

Клиент собран на Qt 6 с использованием QML. Интерфейс разделён на страницы:

- `App.qml` — главное окно, верхняя панель и `StackView` для навигации;
- `MainPage.qml` — логика переходов между разделами;
- `LanguagesListPage.qml` и `ThemesListPage.qml` — выбор курса и темы;
- `TheoryPage.qml` — отображение теории;
- `TaskPage.qml` — редактор решения, запуск и отправка на тесты;
- `ProfilePage.qml` — прогресс пользователя;
- `TeacherMainPage.qml` — формы для добавления учебного контента.

Связь QML с backend проходит через singleton `Api`, зарегистрированный в `main.cpp`. Он создаётся на основе класса `ApiClient` и доступен в QML как модуль `Backend 1.0`.

## Сборка клиента

Требования:

- Qt 6.8 или новее;
- CMake 3.21.1 или новее;
- C++ компилятор, совместимый с Qt 6.

Пример сборки:

```bash
cd client
cmake -S . -B build
cmake --build build
```

После сборки исполняемый файл создаётся в директории сборки. Точное расположение зависит от генератора CMake и операционной системы.

## Настройка backend

Для запуска backend нужен веб-сервер с PHP и MySQL.

Общий порядок:

1. Создать базу данных `Diploma`.
2. Импортировать схему и стартовые данные из `api/db.sql`.
3. Создать пользователя MySQL, от имени которого будет подключаться PHP-код.
4. Убедиться, что путь к файлу с паролем совпадает с тем, который используется в `api/functions.php`:

```php
__DIR__ . "/../../secret/Diploma/dbuser.pswd"
```

5. Разместить каталог `api/` так, чтобы endpoints были доступны по HTTP.
6. При необходимости изменить URL API в `client/App/main.cpp` — сейчас клиент обращается к:

```text
https://diploma.bogaledev.ru/api
```

## Проверка решений

Пользовательское решение отправляется на сервер. Сервер запускает код или проверяет его на тестах, связанных с конкретным заданием. Для каждого задания в таблице `task_tests` хранится вход и ожидаемый выход.

Важно: проверка обычно чувствительна к точному формату вывода — лишние пробелы, отсутствие перевода строки или другой порядок строк могут привести к непройденному тесту.

## Что важно для разработчика

- В проекте уже есть разделение ролей: студент и преподаватель.
- Контент хранится в базе и может расширяться через API.
- Клиентская навигация сосредоточена в QML, а сетевой слой — в `ApiClient`.
- Backend использует Bearer-авторизацию и проверяет права преподавателя для операций создания контента.
- Стартовые учебные данные находятся в SQL-файле, поэтому проект можно быстро развернуть для демонстрации.

## Возможные направления развития

- добавить полноценный редактор кода с подсветкой синтаксиса;
- расширить систему тестирования решений: лимиты времени, памяти, подробные ошибки;
- добавить административную панель для редактирования и удаления контента;
- сделать Docker Compose для быстрого локального запуска backend и базы;
- добавить автоматические тесты для PHP API;
- вынести адрес API в конфигурацию клиента, чтобы не пересобирать приложение при смене сервера;
- улучшить безопасность запуска пользовательского кода через изолированные контейнеры.

## Кратко

Это учебная платформа в формате desktop-приложения: студент изучает программирование и решает задачи, преподаватель управляет материалами, backend хранит контент и проверяет решения. Проект демонстрирует работу с Qt/QML, C++ сетевым слоем, PHP API, MySQL и ролевой моделью пользователей.

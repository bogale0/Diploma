    1. Введение
        1.1. Основание для разработки
            1.1.1. Наименование работы: «Десктопное приложение для онлайн-школы по программированию».
            1.1.2. Основанием для разработки является задание на выпускную квалификационную работу.
            1.1.3. Организация, утвердившая договор: Читинский Педагогический колледж.
        1.2. Назначение и область применения
            1.2.1. Назначение разработки заключается в создании инструмента для обучения программированию, который объединяет теоретический материал и практические задания в одном приложении с безопасным выполнением кода пользователей.
            1.2.2. Основные функциональные задачи приложения включают:
                1.2.2.1. Просмотр теоретических материалов по темам программирования;
                1.2.2.2. Выбор и выполнение практических заданий;
                1.2.2.3. Редактирование и отправка кода на сервер для проверки;
                1.2.2.4. Получение результатов выполнения и обратной связи;
                1.2.2.5. Навигация по курсу обучения.

    2. Требования к программе
        2.1. Требования к функциональным характеристикам
            2.1.1. Работа с курсами и темами:
                2.1.1.1. Возможность загрузки списка тем и заданий с сервера;
                2.1.1.2. Постраничное отображение теоретического материала;
                2.1.1.3. Отслеживание прогресса пользователя по темам и заданиям.
            2.1.2. Редактор кода:
                2.1.2.1. Встроенный редактор с подсветкой синтаксиса;
                2.1.2.2. Возможность редактирования, сохранения и отправки кода на сервер;
                2.1.2.3. Поддержка нескольких языков программирования (в будущем).
            2.1.3. Система проверки решений:
                2.1.3.1. Выполнение пользовательского кода на сервере в изолированной среде Docker;
                2.1.3.2. Контроль ресурсов: память, CPU, время выполнения;
                2.1.3.3. Сравнение результатов с эталонными значениями;
                2.1.3.4. Формирование обратной связи для пользователя.
        2.2. Требования к надёжности
            2.2.1. Требования к обеспечению надежного функционирования программы:
                2.2.1.1. Программа должна продолжать функционировать при сбоях оборудования или сетевого соединения;
                2.2.1.2. Обеспечение сохранности и конфиденциальности данных через аутентификацию и шифрование;
                2.2.1.3. Интуитивно понятный и удобный интерфейс для пользователя без специальных навыков;
                2.2.1.4. Простота сопровождения и расширяемость системы;
                2.2.1.5. Корректное и стабильное выполнение всех функций;
                2.2.1.6. Совместимость с другими программами и операционными системами, поддерживающими стандарты HTTP и JSON.
            2.2.2. Время восстановления после отказа:
                2.2.2.1. При отказе клиентской программы восстановление возможно через перезапуск приложения;
                2.2.2.2. При сбое серверной части восстановление может потребовать нескольких минут до нескольких часов в зависимости от характера ошибки.
            2.2.3. Отказы из-за некорректных действий пользователей системы
                2.2.3.1. Некорректный ввод данных в редакторе кода вызывает уведомление об ошибке;
                2.2.3.2. Попытка выполнения запрещённых действий или превышение прав доступа блокируется системой.

    3. Условия эксплуатации
        3.1. Климатические условия эксплуатации
            3.1.1. Рабочая температура должна находиться в диапазоне от 10 до 35 градусов Цельсия;
            3.1.2. Рабочая влажность должна быть в диапазоне от 30% до 80%.
        3.2. Требования к квалификации и численности персонала
            3.2.1. Требований к квалификации или численности персонала не предъявляется. Программа рассчитана на пользователя без специальных навыков.
        3.3. Требования к составу и параметрам технических средств
            3.3.1. ОС: Windows 10 или выше / Linux;
            3.3.2. Процессор: Intel Core i3 или выше;
            3.3.3. ОЗУ: минимум 4 ГБ;
            3.3.4. Свободное место на диске: 500 МБ;
            3.3.5. Интернет-соединение для синхронизации с сервером.
        3.4. Требования к информационной и программной совместимости
            3.4.1. Требования к информационным структурам и методам решения:
                3.4.1.1. Серверная часть использует базу данных для хранения информации о пользователях, курсах и заданиях;
                3.4.1.2. Клиент работает с API сервера через HTTP-запросы и JSON-ответы;
                3.4.1.3. Структура базы данных должна поддерживать хранение пользователей, ролей, курсов и результатов выполнения заданий.
            3.4.2. Требования к исходным кодам и языкам программирования:
                3.4.2.1. Проект разрабатывается на C++ с использованием Qt Quick для клиента и PHP для сервера;
                3.4.2.2. Код должен быть чистым, структурированным и документированным;
                3.4.2.3. Поддержка open-source разработки (по согласованию).
            3.4.3. Требования к программным средствам, используемым программой:
                3.4.3.1. Поддержка ОС Windows и Linux;
                3.4.3.2. Использование Docker для безопасного выполнения кода;
                3.4.3.3. Интеграция с веб-сервером Apache/Nginx.
            3.4.4. Требования к защите информации и программ:
                3.4.4.1. Система аутентификации пользователей с разграничением прав доступа;
                3.4.4.2. Шифрование конфиденциальной информации;
                3.4.4.3. Ограничение прав и ресурсов при выполнении кода;
                3.4.4.4. Регулярные обновления программного обеспечения.
        3.5. Специальные требования
            3.5.1. Авторизация по логину и паролю;
            3.5.2. Разграничение прав пользователей: студент, преподаватель, администратор;
            3.5.3. Возможность редактирования только допустимых разделов;
            3.5.4. Защита данных при передаче и хранении;
            3.5.5. Регулярное обновление и поддержка системы.

    4. Требования к программной документации
        4.1. Состав программной документации
            4.1.1. Техническое задание;
            4.1.2. Руководство пользователя.

    5. Технико-экономическое обоснование
        5.1. Экономическая эффективность ориентировочно не рассчитывается из-за уникальности требований к проекту.

        7.2. Тестирование
            7.2.1. Проверка всех основных функций приложения;
            7.2.2. Проверка стабильности работы приложения при различных нагрузках;
            7.2.3. Проверка защиты данных пользователей.
        7.3. Приёмка
            7.3.1. Проверка соответствия приложения требованиям безопасности и конфиденциальности данных;
            7.3.2. Проверка оформления приложения в соответствии с корпоративным стилем компании;
            7.3.3. Утверждение результатов тестирования.

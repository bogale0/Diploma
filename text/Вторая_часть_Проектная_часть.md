## 2. ПРОЕКТНАЯ ЧАСТЬ

Тема дипломного проекта: **«Разработка десктопного приложения для онлайн-школы по программированию»**.

Проектная часть описывает разработанное программное средство, его архитектуру, компоненты, реализацию ключевых функций и результаты апробации.

### 2.1. Проектирование программного продукта

#### 2.1.1. Назначение и общая структура

Разрабатываемое приложение предназначено для поддержки процесса обучения программированию в формате онлайн-школы. Продукт обеспечивает:

- просмотр курсов (языков программирования), уроков (тем), теоретических материалов;
- выполнение практических заданий внутри приложения;
- отправку решений на сервер, запуск кода и проверку по автоматическим тестам;
- авторизацию пользователя и хранение прогресса.

Программный продукт построен по клиент‑серверной архитектуре и состоит из следующих компонентов:

- **Клиентское приложение** (`/client`): десктопное приложение на **Qt + QML (Qt Quick Controls)** с С++‑модулем доступа к API;
- **Серверная часть (API)** (`/api`): набор PHP‑скриптов, предоставляющих HTTP‑интерфейс для клиента;
- **База данных** (`/api/db.sql`): схема и стартовые данные (курсы, темы, задания, тесты).

#### 2.1.2. Архитектура клиента

Клиентская часть реализована на Qt с использованием QML‑интерфейсов и навигации через стек страниц. Основные особенности:

- единое окно приложения с пользовательской панелью заголовка и областью контента;
- навигация между страницами («Курсы» → «Уроки» → «Теория» → «Задание») через `StackView`;
- получение данных (курсы/темы/теория/задания) из API и отображение в виде списков и карточек;
- встроенный редактор кода и блоки для запуска/проверки решения.

Также в приложении предусмотрена **роль преподавателя** (панель учителя), позволяющая создавать курсы, темы, задания и тесты через API.

#### 2.1.3. Архитектура серверной части

Сервер реализован на PHP и предоставляет отдельные конечные точки для функций приложения:

- **авторизация и управление сессией**: `login.php`, `signup.php`, `recover_password.php`;
- **получение учебного контента**: `languages.php`, `themes.php`, `theory.php`, `tasks.php`, `task.php`, `progress.php`;
- **выполнение и проверка кода**: `run_task.php`, `check_task.php`;
- **функции преподавателя (создание контента)**: `create_language.php`, `create_theme.php`, `create_task.php`, `create_test.php`.

Выполнение кода организовано через временный каталог на сервере:

- исходный код сохраняется в файле с соответствующим расширением;
- для компилируемых языков запускается компиляция с ограничением по времени;
- запуск выполняется с ограничением по времени, входные данные передаются через `stdin`;
- вывод программы сравнивается с ожидаемым (для проверки по тестам).

В приложении предусмотрена поддержка нескольких языков программирования: **C++**, **Python**, **C#**, **Go**. Выбор языка определяется курсом (языком), к которому относится тема и задание.

#### 2.1.4. Проектирование базы данных

База данных хранит учебный контент, пользователей и результаты обучения. Основные сущности:

- `users`: пользователи (логин, хэш пароля, роль);
- `sessions`: сессии авторизации (bearer‑токен с временем жизни);
- `languages`: курсы по языкам программирования (название, описание, изображение);
- `themes`: темы/уроки, привязанные к языку (`lang_id`), и теоретический материал;
- `tasks`: практические задания, привязанные к теме (`theme_id`);
- `task_tests`: набор тестов для задания (вход/ожидаемый вывод);
- `user_theme_progress`, `user_task_solved`: прогресс прохождения тем и решённых заданий.

Такая структура позволяет расширять систему: добавлять новые языки, темы и задания без изменения логики клиента.

#### 2.1.5. Проектирование пользовательского интерфейса

При проектировании интерфейса был сделан акцент на следующих принципах:

- **видимость контента**: увеличенные шрифты и размеры элементов управления для более комфортного чтения теории и заданий;
- **последовательная навигация**: пользователь всегда понимает, где находится, и может вернуться на шаг назад;
- **разделение теории и практики**: теория отображается отдельной страницей, задания — отдельной страницей с редактором;
- **оперативная обратная связь**: после запуска кода и проверки по тестам пользователь получает статус выполнения.

### 2.2. Руководство пользователя

#### 2.2.1. Запуск приложения

После запуска приложения отображается стартовый экран, затем открывается главная страница с навигацией.

#### 2.2.2. Выбор курса и урока

1. На странице **«Курсы»** пользователь выбирает интересующий язык программирования.
2. На странице **«Уроки»** пользователь выбирает тему.

#### 2.2.3. Изучение теории

На странице **«Теория»** отображается материал по выбранной теме. Внизу страницы находятся кнопки **«Задание 1», «Задание 2», …**, которые открывают практическую часть по соответствующему заданию.

#### 2.2.4. Выполнение задания

На странице **задания** доступны:

- поле редактора кода;
- блок «Пример» (публичные пример входа/выхода — при наличии);
- блок «Свой тест», где пользователь вводит собственные входные данные и запускает программу;
- кнопки:
  - **«Запустить»** — выполнить программу на введённом пользователем входе;
  - **«Отправить»** — проверить решение по набору автоматических тестов.

Результат отображается в блоке результата:

- «Успешно» и вывод программы при корректном запуске;
- «Ошибка компиляции»/«Ошибка выполнения» при проблемах;
- «Верное решение» или «Неправильный ответ» при проверке по тестам.

#### 2.2.5. Авторизация и прогресс

Для выполнения проверяемых заданий требуется авторизация. После входа пользователь получает доступ к странице **«Личный кабинет»**, где отображаются:

- общий прогресс по курсам;
- прогресс по каждому курсу;
- возможность выхода из аккаунта.

#### 2.2.6. Функции преподавателя

Пользователь с ролью **teacher** получает доступ к панели учителя, где может:

- создавать курсы (языки);
- добавлять темы и теорию;
- создавать задания и тесты.

### 2.3. Результаты апробации

#### 2.3.1. Цель апробации

Апробация проводилась для подтверждения работоспособности следующих функций:

- авторизация и получение данных из API;
- отображение списка курсов и тем;
- получение теоретических материалов;
- запуск пользовательского решения;
- проверка решения по тестам и фиксация прогресса.

#### 2.3.2. Проведённые испытания

В ходе проверки были выполнены действия:

- проверка переходов между страницами и корректности отображения контента;
- запуск кода с пользовательским вводом и анализ вывода;
- проверка по тестам на заданиях с корректными и некорректными решениями;
- проверка сохранения результата при успешном решении задания и обновления прогресса;
- проверка отображения увеличенных шрифтов и корректности лэйаута после изменения размеров элементов интерфейса.

#### 2.3.3. Итоги

По результатам апробации подтверждено:

- приложение корректно загружает и отображает учебные данные (курсы, темы, теория, задания);
- серверная часть выполняет и проверяет решения, формируя понятную обратную связь;
- поддерживается выполнение задач на **C++**, **Python**, **C#**, **Go** (в рамках настроенных на сервере инструментов);
- пользовательский интерфейс обеспечивает более заметное отображение контента за счёт увеличенных шрифтов и размеров элементов.

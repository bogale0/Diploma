### 2.2. Руководство пользователя

#### 2.2.1. Общие сведения о пользовательском интерфейсе

Пользовательский интерфейс приложения «Школа программирования» реализован в виде единого окна с последовательной навигацией по учебному сценарию: выбор курса → выбор темы → изучение теории → выполнение задания. В верхней части окна всегда доступна пользовательская кнопка профиля (👤) для быстрого перехода в личный кабинет и кнопка закрытия приложения (✕). Сразу после запуска отображается заставка, затем открывается основная рабочая область (Рисунок 1). [Screenshot of SplashScreen.qml]

#### 2.2.2. Главное окно и список курсов

После завершения заставки пользователь попадает на первую рабочую страницу — список курсов (языков программирования). Каждый элемент списка содержит название и описание курса; выбор курса выполняется нажатием на карточку элемента. Эта страница является стартовой точкой пользовательского пути (Рисунок 2). [Screenshot of LanguagesListPage.qml]

#### 2.2.3. Окно списка тем выбранного курса

После выбора курса открывается окно со списком тем. На странице отображаются все темы по выбранному языку программирования; переход к следующему шагу выполняется выбором нужной темы. Для удобства каждая тема сопровождается кратким пояснением (Рисунок 3). [Screenshot of ThemesListPage.qml]

#### 2.2.4. Окно «Теория»

После выбора темы открывается окно теоретического материала. В верхней части отображается текст теории, а ниже — кнопки перехода к практическим заданиям темы (например, «Задание 1», «Задание 2» и т.д.). Выбор задания выполняется нажатием соответствующей кнопки (Рисунок 4). [Screenshot of TheoryPage.qml]

#### 2.2.5. Окно авторизации

Если пользователь пытается открыть задание или личный кабинет без входа в систему, приложение автоматически открывает окно авторизации. В этом окне доступны стандартные режимы: вход, регистрация и восстановление пароля. После успешной авторизации пользователь возвращается к запрошенному ранее действию (Рисунок 5). [Screenshot of Authorization.qml]

#### 2.2.6. Окно выполнения задания

Окно задания содержит все основные элементы практической работы:

- текст условия задания;
- поле редактора кода;
- блок публичного примера (вход/выход);
- поле «Свой тест» для ввода пользовательских данных;
- кнопки:
  - **«Запустить»** — запуск решения на пользовательском вводе;
  - **«Отправить»** — проверка решения на автоматических тестах.

Ниже выводится результирующий блок со статусом выполнения (успешный запуск, ошибка компиляции/выполнения, число пройденных тестов, принятие решения). Таким образом, пользователь получает оперативную обратную связь после каждого действия (Рисунок 6). [Screenshot of TaskPage.qml]

#### 2.2.7. Окно «Личный кабинет»

Окно личного кабинета открывается по кнопке профиля (👤) после авторизации. Здесь отображаются:

- общий прогресс обучения в процентах;
- прогресс по каждому курсу в виде полосы заполнения;
- числовой показатель пройденных и доступных уроков;
- кнопка **«Выйти из аккаунта»** для завершения сессии.

Страница используется для мониторинга результатов обучения и управления текущим входом пользователя (Рисунок 7). [Screenshot of ProfilePage.qml]

#### 2.2.8. Окно «Панель учителя»

Если аккаунт имеет роль **teacher**, вместо личного кабинета открывается окно «Панель учителя». Вкладки панели предоставляют инструменты администрирования учебного контента:

- **«Новый курс»** — создание нового курса (языка);
- **«Новая тема»** — добавление темы и теоретического материала;
- **«Новое задание»** — создание практического задания;
- **«Новый тест»** — добавление теста для автоматической проверки;
- **«Выйти»** — выход из учётной записи преподавателя.

Этот интерфейс обеспечивает полный цикл наполнения системы учебными материалами без перехода во внешние административные инструменты (Рисунок 8). [Screenshot of TeacherMainPage.qml]

### 2.3. Результаты апробации

#### 2.3.1. Цель апробации

Апробация проводилась для подтверждения работоспособности следующих функций:

- авторизация и получение данных из API;
- отображение списка курсов и тем;
- получение теоретических материалов;
- запуск пользовательского решения;
- проверка решения по тестам и фиксация прогресса.

#### 2.3.2. Проведённые испытания

В ходе проверки были выполнены действия:

- проверка переходов между страницами и корректности отображения контента;
- запуск кода с пользовательским вводом и анализ вывода;
- проверка по тестам на заданиях с корректными и некорректными решениями;
- проверка сохранения результата при успешном решении задания и обновления прогресса;
- проверка отображения увеличенных шрифтов и корректности лэйаута после изменения размеров элементов интерфейса.

#### 2.3.3. Итоги

По результатам апробации подтверждено:

- приложение корректно загружает и отображает учебные данные (курсы, темы, теория, задания);
- серверная часть выполняет и проверяет решения, формируя понятную обратную связь;
- поддерживается выполнение задач на **C++**, **Python**, **C#**, **Go** (в рамках настроенных на сервере инструментов);
- пользовательский интерфейс обеспечивает более заметное отображение контента за счёт увеличенных шрифтов и размеров элементов.
